# Product Requirements Document - ClickTool

## 1. 项目概述
ClickTool 是一个 Windows 桌面悬浮工具，用于录制鼠标操作、管理多个执行方案，并支持整段回放或按“步”回放。

## 2. 当前核心功能
### 2.1 悬浮 UI
- 默认状态是圆形悬浮按钮。
- 左键单击圆形按钮：直接播放当前方案。
- 左键拖动圆形按钮：移动悬浮窗位置。
- 右键单击圆形按钮：展开矩形控制面板。
- 展开态支持拖曳空白区域移动窗口。
- 展开态支持拖曳边缘调整窗口大小。
- 标题栏支持一键置底，再次点击恢复置顶。

### 2.2 录制与回放
- 支持全局鼠标移动、按下、抬起录制。
- 支持整段播放。
- 支持循环播放，直到用户点击停止按钮或触发停止热键。
- 支持按步播放。
- 默认将“鼠标抬起”视为一个步骤结束点。

### 2.3 步骤编辑
- 步骤相关操作统一在“打开步骤列表”窗口中完成。
- 支持步骤折叠/展开查看。
- 支持单条删除、批量删除、合并为一步、逐条分步、切换步尾。
- 支持编辑单条记录的坐标值和等待时间。
- 支持复制单个步骤。
- 步骤列表窗口顶部会说明 3 个数字框分别对应 X 坐标、Y 坐标和等待时间(ms)。

### 2.4 方案管理
- 方案相关操作统一在“打开方案列表”窗口中完成。
- 支持查看全部方案。
- 支持勾选切换当前执行方案。
- 支持修改当前方案名称。
- 支持将当前方案另存为新方案。
- 支持删除当前方案。

### 2.5 热键与设置
- 支持窗口查看热键。
- 支持录制热键。
- 支持播放热键。
- 支持按步播放热键。
- 支持停止热键。
- 热键设置时，按 `Esc` 代表不设置该热键。
- 支持悬浮窗透明度调节，范围为 5%-100%，并且是连续变化。

### 2.6 打包与图标
- 应用使用自定义圆形播放图标。
- 单文件发布时使用 `Assets/icon.ico` 作为应用图标。

## 3. 当前交互定义
- “步”是若干条连续操作组成的逻辑组。
- 每个步骤以“步尾”结束。
- 主功能面板不再直接展示步骤列表。
- 主功能面板不再直接展示方案列表。
- 方案列表窗口用于方案切换和方案管理。
- 步骤列表窗口用于步骤查看和步骤编辑。

## 4. 当前实现重点
- 主窗口交互已拆分为多个 partial 文件，便于继续维护。
- 录制文件编辑后会覆盖保存回原方案文件。
- 已有两个独立交互窗口：
  - 步骤列表窗口
  - 方案列表窗口
