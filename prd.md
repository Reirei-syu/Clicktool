# Product Requirements Document - ClickTool

## 1. 项目概述
ClickTool 是一个面向 Windows 的桌面悬浮工具，用于录制鼠标操作、管理多个执行方案，并支持整段播放、循环播放和按步播放。

## 2. 当前核心功能
### 2.1 悬浮 UI
- 默认状态为圆形悬浮播放按钮。
- 左键单击圆形按钮：直接播放当前方案。
- 左键按住圆形按钮：拖动悬浮窗位置。
- 右键单击圆形按钮：展开矩形主功能面板。
- 展开态窗口支持拖动空白区域移动。
- 展开态窗口支持拖动边缘调整大小。
- 标题栏支持窗口置底，再次点击恢复置顶。

### 2.2 录制与播放
- 支持全局鼠标移动、按下、抬起录制。
- 支持播放全部。
- 支持循环播放。
- 循环次数支持手动输入，范围为 `1-*`，其中 `*` 表示无限循环。
- 支持按步播放。
- 停止按钮和停止热键都可以中断普通播放或循环播放。

### 2.3 步骤编辑
- 主功能面板不直接显示步骤列表。
- 所有步骤编辑统一在“打开步骤列表”窗口中完成。
- 支持步骤折叠/展开查看。
- 支持单条删除、批量删除、合并为一步、逐条分步、切换步尾。
- 支持编辑单条记录的 X、Y 和等待时间(ms)。
- 支持复制单个步骤。
- 支持批量复制选中步骤。
- 支持批量修改选中记录的等待时间。
- 顶部说明文字会解释 3 个数字框分别对应 X 坐标、Y 坐标和等待时间。
- 除鼠标移动外的操作记录使用黄色文字高亮，便于区分。

### 2.4 步骤定义
- 新录制完成后，所有记录默认处于“未分组”状态，即动作默认显示为“步骤中”。
- 如果用户尚未显式定义步尾，系统会把整段录制默认归集到“步骤 1”中。
- 此时所有记录都显示为“步骤中”，按步播放一次会执行完整的默认步骤。
- 用户可通过“合并为一步”把连续记录显式组合成一个步骤。

### 2.5 方案管理
- 方案相关操作统一在“打开方案列表”窗口中完成。
- 支持查看全部方案。
- 支持勾选切换当前执行方案。
- 支持修改当前方案名称。
- 支持将当前方案另存为新方案。
- 支持删除当前方案。

### 2.6 热键与设置
- 支持窗口查看、录制、播放、按步播放、停止五类热键。
- 热键支持单键，或“一个修饰键 + 一个主键”的双键组合。
- 当前支持的修饰键为 `Ctrl`、`Alt`、`Shift`、`Win`。
- 按 `Esc` 表示清空该热键。
- 不允许热键重复。
- 支持悬浮窗透明度设置，范围为 5% 到 100%，且为连续变化。

### 2.7 打包与图标
- 应用使用自定义圆形播放图标。
- 单文件发布使用 `Assets/icon.ico` 作为应用图标。
- 已支持生成 MSI 安装包，安装时可选择安装路径。

## 3. 当前交互定义
- 主功能面板只保留执行控制、方案入口、步骤入口、摘要信息和热键设置。
- 方案列表窗口负责方案切换与方案管理。
- 步骤列表窗口负责步骤查看与步骤编辑。
- 循环播放的次数输入位于主功能面板播放区下方。
- 步骤列表窗口中的批量操作基于当前勾选记录执行。

## 4. 当前实现重点
- 主窗口已拆分为多个 `partial` 文件，便于继续维护。
- 录制文件编辑后会覆盖保存回原方案文件。
- 新录制默认不自动写入步尾定义。
- 步骤系统同时支持显式分组和“整段默认归入步骤 1”的初始状态。
- 已提供独立的步骤列表窗口与方案列表窗口。
